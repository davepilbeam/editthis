    Counter Module

    Displays an Up or Down Numeric Count.

    Configuration:

    Add the following data-attributes to a div with a class of text inside an editmodule with a class of counter

    data-start = '0kg': [ sets the starting number (add the unit extension if needed) ] REQUIRED - make sure your target paragraph text is identical

    data-end = '9000': [ sets the end number (no extension is required) ] REQUIRED

    data-interval = '100': [ sets the time between each count in milliseconds ] (the default is 100ms)

    data-target = '2': [ sets the number of the target paragraph within the parent module ] (the default is paragraph 1)

    data-units = '0.01': [ sets the amount to count up each time ] (the default is 1)

    Form Module

    Displays a Form.

    Configuration:

    form id = cgi_form_(0+) where 0+ is the number of the Forms on the page when it is first created

    form response target = the first page element with id="cgi_form_X_result" will become the target of the Form result otherwise the form will target itself

    Adding opt_ = before the field name means it is optional

    Adding pre_ = before the field name means it is required

    _(0+) = should be added to the end of all field names and ids in form 0+ ie: the number of that Form on the page

    Adding a class of tt_focus to a field will empty that field on focus then restore the field's value if empty on blur

    Add the following hidden fields to a Form inside an editmodule with a class of "form"

    cgireturn = if present and has value adds its contents to the server's response

    cgiurl = if present, with value 0 or 1 or 2, it will post the form's data directly to the Form's action url and does not return a response to the Page - (0 with validation on names starting with 'pre_', 1 removes validation, 2 enforces validation on all )

    copytype = if present and has value sends a copy of the email to the Form email field address

    destination = if present and has value uses the Form's action url as the cgi destination but unlike using cgiurl returns the AJAX response to the Page as usual.

    formtype = becomes the Form subject (the default is "Email Enquiry"). This is used in defs.pl to configure aspects of the Form eg: defining a group to copy an email to = $COPY{'Email Enquiry'}.See_defs.pl)

    html = if present and has value REMOVES html format from email ie (default is htm; formatting)

    locate = Used in conjunction with cgiurl will define a page/document href to forward to after Form submission.

    local_js = if present Form will fire local javascript functions: (see below, G.forms[name of Form] and tt_local class)

    recipients = if present and has value sends copies to recipients in named group (see definitions in cgi-bin/defs.pl)

    A button with class of 'tt_back' will also submit form to url if the Form field input[id^=destination_] (for 'back' action)

    Spam

    If any field within a li with a class of manifest contains a value (entered by bot), the Form submission will fail. The input name must end in '-manifest' eg: address-manifest_0

    Toggle Field View

    Add view_[id] to an option_value/input and it will toggle visibility of the target ul or li element with id view_[id] if it exists in the Form. NB: slashes in values turn to dashes in ids.

    Local Javascript

    An input with name="opt_local_js1" id="local_js1" will send it's value to a local function at G.forms[name_of_this_Form]

    If javascript is disabled, the Form uses cgi to send to next Form (returns a new static page) with all fields added as hidden inputs on next Form.

    Elements with data-$fieldname / url(data-$fieldname) will be replaced on the next returned Form.

    Library Module

    Displays documents from the site's Digital Library.

    Configuration:

    Add class libraryarea to an ul containing an editmodule with a class of library

    Add the following data-attributes to the link inside: <div class="text"><a></a></div> or pass them using using view eg: data-pass="sort:21,title:Resources"

    data-amount = '0': [ switches on the mins heights in mobile view ] (the default is auto heights in mobile view)

    data-basefolder = 'documents/Partner-Portal' [ sets the defined Documents folder as the Library Source ] (the default is defined in G.library in config.js )

    data-depth = '1': [ defines a depth level to display folders as open in accordion ] (the default is '0' which is none open)

    data-form = 'on': [ enables a required email download form (manually add form as below if required) ] (the default is no form required)

    data-id = 'paginate': [ switches on pagination for the data response format if applicable ] (the default is no pagination)

    data-names = 'editarea,editfocus': [ defines tags on which to filter results (separate multiples with a comma) ]

    data-onlyarea = 'me' [ will only show matching 'area' tag results if present ]

    data-onlyfocus = 'me': [ will only show matching 'focus' tag results if present ]

    data-sort = '21': [ defines sort order of response data: alpha:AZ or reverse alpha:ZA or date:'12' or reverse date:'21' ] (the default is 21: reverse date)

    data-title = '': [ includes this field as a title to the returned Library html ] (the default is no title)

    data-values = 'security,z system': [ defines tag values (should match data-names) by which to filter results ( separate multiples with a comma) ]

    To use Dropbox as the Library Source, change the action of the form from 'email.pl' to 'dropbox.pl' and configure the following settings as appropriate in G.library in config.js

    dropbox: { url:'dropbox.pl',appname:'thatsthat_library',id:'list',sname:'Online Derby',thumbfolder:'documents/Dropbox/' }

    To add a data-code attribute to a libraryblock element add [etc] to the filename eg: new-leaflet[A4].pdf then configure the css as appropriate in library.css

    To display any added [data-code] text in a file display name eg:new leaflet[A4] set nocode to null in G.library.nocode in config.js

    Social Media Feed Module (RSS)

    Displays an RSS Feed within a Page.

    Configuration:

    Add the following data-attributes to an a link inside a div with a class of text inside an editmodule with a class of rss

    data-characters = '200': sets number of characters to show NB: if data-characters == 0 and data-html == 'off', then only entry titles are shown ] (default is in G.rss.limit in config.js)

    data-full = 'on': [ overrides data-character limit ] used to include all hidden images from received Blogger data (default is off)

    data-html = 'on': [ include html ] (default is on)

    data-posts = '3': [ sets number of posts to show ] (default is in G.rss.posts in config.js)

    data-rules = 'off': [ adds rules between posts ] (default is off)

    Usage:

    If not working on client's server Twitter and Facebook scripts can be configured to relay from thatsthat.uk by setting G.server to 'pe' in config.js

    Configure the display styles in G.rss in config.js. Use this to switch icon or byline display off etc.

    Setting a character limit will not affect text within any html tags.

    Remember to use &amp;in the href urls!

    Spaces in blogger labels should be '%20' = http://karlsonuk.blogspot.com/feeds/posts/default/-/News%202013"

    Slideshow Module

    Animates local or remote elements using CSS Transforms. Cross-domain compatible.

    Configuration:

    Add the following data attributes to an ul.area or pass them in a view module using view eg: data-pass="controller:on,transition:fadein"

    Area Attributes

    data-auto = 'on': [ enables autostart ] (the default is off)

    data-controller = 'on': [ controls whether controller and pause button is visible and overrides any autostart. ] The default is off.

    data-start = '0': [ defines which Slide to display at start ] (the default is Slide 0)

    data-delay = '3000': [ specifies the default Slide transition delay in milliseconds ] (the default is 3000ms)

    data-interval = '1500': [ specifies the default Slide transition duration in milliseconds ] (the default is 1000ms via a class of 'css-1s')

    data-startdelay = '3000': [ controls a one-time delay before the first Slide transition starts in milliseconds ] this is useful when an initial slideholder is used (the default is no delay)

    data-transition = 'none': [ defines a global Slide CSS3 transition from desktop.js ] (the default is no transition)

    data-showholder = 'on': [ defines whether to display the inital slideholder and defer loading the first slide ] (the default is off - slideholder only appears if no javascript or unsupported browser)

    data-keepholder = 'on': [ defines whether to delete or preserve any initial slideholder after the first Slide is displayed ] (the default is off)

    data-key = 'on': [ adds keys to a controller (if present) derived from adding a 'data-slidekey' attribute to a slide ] (the default is off)

    Slide Attributes

    data-delay = '3000': [ controls the delay before the next sibling Slide transition will start in milliseconds ] (the default is the global delay)

    data-hold = '6000': [ defines an optional additional transitional delay in milliseconds which is added to any data-delay ] (the default is none)

    data-interval = '1000': [ specifies a Slide's transition duration in milliseconds, set from classes in G.slideshow.kf in config.js ] (the default is the global interval)

    data-transition = 'none': [ defines a Slide's CSS3 transition from G.slideshow.trans in config.js ] (the default is the global transition)

    data-slidekey = 'Slide 2': [ defines a key for the Slide to be added in the controller ] (the default is 'Slide X' if keys are enabled or blank if not)

    Element Attributes

    data-delay = '3000': [ controls the delay before an Element's transition starts in milliseconds ] A negative number will run the animation from that number in instead of at 0. Set this to the negative of the element's interval for instant animation. (the default is the parent Slide delay)

    data-hold = '6000': [ defines an optional additional transitional delay in milliseconds which is added to any next data-delay ] (the default is none)

    data-interval = '1000': [ specifies a Slide's transition duration in milliseconds, set from classes in G.slideshow.kf in config.js ] (the default is the parent Slide interval)

    data-transition = 'none': [ defines a Slide's CSS3 transition from G.slideshow.trans in config.js ] (the default is the parent Slide transition)

    data-hide = '1000': [ defines how long to wait in milliseconds before hiding an element after its transition ] (the default is not to hide)

    data-slide = 'slideup' or 'slidedown' or 'slideleft' or 'slideright': [ activates a Slider Module to any '.edittext' element on hover using the supplied CSS class ] (the default is no Slider)

    Usage:

    Add an inital style of opacity:1 to any slideshow slides or elements that need to be visible if CSS3 is not supported but invisible if it is (as all style attributes are removed during slideshow setup).

    Depending on your slide transitions, you might need to add a first holding slide with a class of 'slideholder' which is deleted after the first proper slide is displayed.

    Controller bar styles are in Slideshow css in desktop.css

    Transitions for data-transition are in desktop.js

    none

    fadeout

    fadein

    slidedown

    slideleft

    slideright

    slideup

    slidewall

    slidefullright

    slidefullleft

    slidefullup

    slidefulldown

    slideswing

    - The Module will add data-list="s0+" attributes to each Slide and Element (counting upwards) if not already present in the html page.

    - Images will require a '_mobile' version which is swapped out if the viewport is mobile-sized and the Slideshow is the result of a pulled View.

    HINT: To easily position an inner element for transition add css position/padding that are OPPOSITE to its css class eg: bottom:10px; left:10%; padding-bottom:400px; for a class of css-slideup. The padding makes the element taller or wider which in turn makes the elements 100% XY transition greater.

    HINT: If a sliding transition element seems to simply appear without sliding, try increasing its transition time (data-interval) to give the element more display time on screen.

    HINT: If required use 'data-hold' to delay the next slide - this is particularly useful when delaying the reappearance of the first slide after the last slide.

    View Module

    Inserts elements from editable Pages within a Page. Cross-domain compatible.

    Configuration:

    Add the following data-attributes to the link inside: <div class="text"><a></a></div>

    data-amount = '1': [ defines number of results to show and paginate if data-id is 'index' (the default is 0 to show all) ]

    data-delay = 'on': [ if present will defer loading so that an external link can be used to activate ] (the default is load on page load)

    data-empty = 'a message': [ if data-id is filelist, this message will display if no documents are returned ] (the default is no message)

    data-exclude = 'me': [ will remove requesting page from results if present ]

    data-filter = 'editarchive': [ defines filter tag of results to show and paginate if data-id is 'archive' (the default is to show all) ]

    data-format = 'stacker': [ will format elements as a Stacker module ] or 'swiper': [ will format elements as a Swiper module ] or 'slideshow': [ will format elements as a Slideshow module ] or 'library' [ will format elements as a Library module ] or 'lightbox' [ will format elements as a Lightbox module ] (the default is no format) NB: use data-pass to specify any attributes to be applied to the final module

    data-full = '1': [ defines the number of elements included within a pulled block ] (the default is 1 ie: the first editimage element and the first edittext element)

    data-id = 'index': [ show paginated page list from linked Index page ] or 'list': [ show page list from linked Index page ] or 'menu': [ show Menu listing from linked page (for sitemap use 'Site-Map.html' / for site menu use 'index.html') ] or 'images': [ show images from linked page or folder ] or 'blocks': [ show Editblock areas from linked page ] or 'area': [ show Area from linked page ] or 'archive': [ shows menu from ARCHIVE folder ] or 'filelist': [ shows list of files from linked folder ] REQUIRED

    data-link = on': [ will disable links on pulled images ] (the default is 'on' to enable links)

    data-names = 'editarea,editfocus': [ defines tags on which to filter results (separate multiples with a comma) ] use 'classname' if ul.area classname is required (eg slideshowarea)

    data-onlyarea = 'me' [ will only show matching 'area' tag pages from results if present ]

    data-onlyfocus = 'me': [ will only show matching 'focus' tag pages from results if present ]

    data-pass = 'columns:3,autoplay:on' [ will pass on the included data attributes if a module is specified in data-format ]

    data-scroll = 'on': [ if enabled will scroll down to any pulled Stacker ] ( the default is no scroll )

    data-sort = 'unrank' [ display by descending menu rank ] or 'rank' [ display by ascending menu rank ] or '21': [ display latest (pages by editdate tag or page mod date / files by date tag or file creation date) first ] or '12': [ display earliest first ] or 'az': [ display alphanumeric ] or 'za': [ display reverse alphanumeric ] (the default is by menu rank)

    data-start = 'random': [ will randomise return order if present ]

    data-submenu = 'on': [ will allow hidden subpages to be visible in a menu listing ] (the default is not to list hidden subpages)

    data-values = 'security,z system': [ defines tag values (should match data-names) or ul.area classname (eg: 'slideshowarea') by which to filter results (separate multiples with a comma) ]

    data-wrap = 'none': [ doesn't wrap returned data ] or 'list': [ wraps returned non-menu data in 'li' elements ] or 'pull': [ wraps returned non-menu data in 'div' elements ] (the default is no wrap)

    Usage:

    To allow for non-Javascript users, non-folder urls must be specified as follows: [full cgi path]/view.pl?url=[page] eg: ../cgi-bin/view.pl?url=News.html&id=menu ( add '&format=fullmenu' to force a full menu listing when required )

  • Configuration Variables

    E.scrolladj will adjust the height of the autoscroll when an Accordion is opened. Alter in config.js (the default is 0)

    $dositemap switches on auto updating of the sitemap.xml file when set to '1'. Alter in defs.pl (the default is 'undef')

    %headers => analytics_gref if required sets the Google Code on new Pages when formatted as 'analytics_gref' => [ "name","content","UA-XXXXXXXX-1","Google Code" ]'. Alter in defs.pl (the default is no code)

    %headers => analytics_lref if required sets the Lead Forensics Code on new Pages when formatted as 'analytics_lref' => [ "name","content","domainname.co.uk","Lead Forensics Code" ]'. Alter in defs.pl (the default is no code)

    %headers => analytics_wref if required sets the WOW Code on new Pages when formatted as 'analytics_wref' => [ "name","content","49accf29-f990-4afc-8cb3-d248d186edf7","WOW Code" ]'. Alter in defs.pl (the default is no code)
